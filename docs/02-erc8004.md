# 02 · ERC-8004 identity + reputation

**Goal:** give your agent a portable onchain identity (an ERC-721) and a
readable reputation trail, then read both back.

**Files:** [`src/lib/erc8004.ts`](../src/lib/erc8004.ts),
[`src/components/IdentityCard.tsx`](../src/components/IdentityCard.tsx),
[`src/components/ReputationWriter.tsx`](../src/components/ReputationWriter.tsx)

## What ERC-8004 is

ERC-8004 ("Trustless Agents") standardizes agent identity and reputation as
onchain primitives:

- **Identity Registry** — `register(agentURI)` mints your agent an ERC-721 and
  returns an `agentId`. The `agentURI` points at an off-chain JSON "agent card"
  describing the agent.
- **Reputation Registry** — `giveFeedback(...)` writes a signed score about an
  agent. Track records become reconstructable from chain state alone.

Arke proved this end-to-end with **agent #20360** on Arc testnet — that ID is
the default in the read demo so you have something real to query.

## The exact reputation signature (get this byte-perfect)

```
giveFeedback(uint256,int128,uint8,string,string,string,string,bytes32)
```

| arg | type | meaning |
|---|---|---|
| `agentId` | `uint256` | which agent the feedback is about |
| `value` | `int128` | the score — **signed**, so it can be negative |
| `decimals` | `uint8` | decimal places for `value` (950 + decimals 2 → 9.50) |
| `tag1`–`tag4` | `string` | free-form labels (e.g. `"accuracy"`) |
| `feedbackHash` | `bytes32` | commitment to off-chain evidence, or `0x0` |

A single mismatched type changes the 4-byte function selector and the call
reverts with no useful reason. Copy the signature from the Arc docs
"Register your first AI agent" tutorial exactly — which is what
[`erc8004.ts`](../src/lib/erc8004.ts) does.

## ⚠️ The discoverability gotcha

The **Identity Registry** sits at a recognizable vanity address starting with
`0x8004…`, so it's easy to find:

```
0x8004A818BFB912233c491871b3d84c89A494BD9e
```

The **Reputation Registry** is *not* documented right next to it. We sourced
its Arc address from the **erc-8004 contracts repo**, not the main tutorial:

```
0x8004B663056A597Dffe9eCcC1965A193B7388713
```

Both are hardcoded in `erc8004.ts` so you don't lose an hour hunting. ERC-8004
is still evolving — re-verify against the live contract on Arcscan before you
rely on these in production.

## Fork just this

Copy `src/lib/erc8004.ts` plus the two components. They depend only on wagmi,

identity + reputation surface for any agent.

## The self-attestation pattern

`ReputationWriter` hashes a human note into `feedbackHash` and writes it with a
tag. Storing the *hash* (not the text) keeps gas low while letting you prove
later what the feedback referred to. Because `value` is signed, honest ledgers
can record penalties as well as praise.
